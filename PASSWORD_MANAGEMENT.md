## 使用流程

### 手动分配模式（当前）

1. **用户付款** → 微信/支付宝收款
2. **人工审核** → 确认收款后从列表选择一个密码
3. **发送密码** → 通过微信/邮件发送给用户
4. **更新状态** → 在此文件标记为"已使用"
5. **添加有效期** → 在 `premium/index.html` 的 `PASSWORDS` 配置中添加密码（有效期30天）

### 配置更新方法

在 `premium/index.html` 中找到密码配置部分，添加新密码：

```javascript
const PASSWORDS = {
    'DIAOYAN2025': { tier: '调研版', level: 1 },
    'ADMIN2025': { tier: '管理员', level: 3 },
    // 新增用户密码（示例）
    'a8Kj2mP9xQ': { tier: '调研版', level: 1, expiry: '2026-05-17' },
};
```

## 自动付费技术方案

由于当前部署在 **GitHub Pages（纯静态托管）**，实现自动付费有以下几种方案：

### 方案1：第三方表单+人工审核（推荐，零成本）

**流程：**
```
用户点击"购买" → 跳转到金数据/麦客表单 → 填写信息+上传付款截图 
→ 你收到通知 → 手动发送密码
```

**优点：** 无需服务器，免费使用  
**缺点：** 需要人工介入

### 方案2：收款码自动展示（最简单）

在付费页面上直接展示：
- 微信收款码（¥39.9）
- 支付宝收款码（¥39.9）
- 附言要求：填写用户手机号/邮箱
- 承诺：付款后2小时内发送密码

### 方案3：接入Stripe/支付宝（需要后端）

**问题：** GitHub Pages 不支持后端代码运行

**解决方案：**
- 使用 **Vercel Serverless Functions** 或 **Cloudflare Workers** 作为支付网关
- 用户付款成功后，自动调用API发送密码（邮件/短信）

**成本：** 每月约 $0-5（按调用次数计费）  
**开发时间：** 1-2天

### 方案4：飞书/钉钉机器人（折中方案）

1. 用户在网页点击"购买" → 弹出收款码
2. 用户付款后截图 → 发送到飞书/钉钉群
3. 你收到通知 → 在群里回复密码（或私聊发送）
4. 可同时存档用户信息
